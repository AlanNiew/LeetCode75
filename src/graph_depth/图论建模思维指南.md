# 图论建模思维指南

## 一、什么是图论建模？

**核心思想**：将实际问题中的"对象"抽象为**节点（顶点）**，将对象之间的"关系"抽象为**边**。

```
现实问题 → 识别对象和关系 → 构建图模型 → 应用图算法 → 得到答案
```

---

## 二、如何识别题目可以用图？

### 🔍 关键信号词

当你看到以下词汇时，应该立即想到图：

| 信号词 | 对应图概念 | 示例 |
|--------|-----------|------|
| **关系** | 边 | A 是 B 的朋友 |
| **依赖** | 有向边 | A 依赖于 B |
| **连接** | 边 | 城市 A 到城市 B 有公路 |
| **传递** | 路径 | A 认识 B，B 认识 C → A 通过 B 认识 C |
| **网络** | 图 | 社交网络、计算机网络 |
| **可达性** | 路径搜索 | 能否从 A 到达 B |
| **最短/最少** | 最短路径 | 最少经过几个人能找到目标 |

### 🎯 问题特征

**满足以下任一特征，很可能用图：**

1. ✅ 涉及多个对象之间的关系
2. ✅ 需要从一个状态转移到另一个状态
3. ✅ 关系可以传递或组合
4. ✅ 问"是否可达"、"最少步骤"、"所有可能路径"
5. ✅ 有明确的起点和终点

## 三、经典建模案例

### 案例 1：除法求值（LeetCode 399）⭐

#### 题目
```
已知：a/b = 2.0, b/c = 3.0
求：a/c = ?
```

#### 建模过程

**传统思维（错误）**：
- 这是数学题，需要代数推导
- 要分类讨论：正向、反向、组合...

**图思维（正确）**：
```
1. 识别对象：变量 a, b, c
2. 识别关系：除法公式 a/b=2.0 表示 a 和 b 有关系
3. 建图：
   - 节点：{a, b, c}
   - 边：a→b (权重 2.0), b→a (权重 0.5)
         b→c (权重 3.0), c→b (权重 0.333)
4. 问题转化：
   - 求 a/c → 找从 a 到 c 的路径
   - 结果 = 路径上所有权重的乘积
```

#### 代码框架
```java
// 建图
Map<String, Map<String, Double>> graph_breadth = new HashMap<>();
for (每个公式 a/b = value) {
    graph_breadth.get(a).put(b, value);      // 正向边
    graph_breadth.get(b).put(a, 1.0/value);  // 反向边
}

// BFS 找路径
double bfs(String start, String target) {
    Queue<Pair<String, Double>> queue;
    // pathValue 累积乘积
    while (!queue.isEmpty()) {
        if (current.equals(target)) return pathValue;
        for (邻居 neighbor : graph_breadth.get(current)) {
            queue.offer(neighbor, pathValue * weight);
        }
    }
}
```

#### 举一反三
**类似问题**：
- 货币兑换：USD/CNY=7.0, CNY/JPY=0.05 → USD/JPY=?
- 单位换算：1 米=100 厘米，1 厘米=10 毫米 → 1 米=?毫米

### 案例 2：课程表（LeetCode 207）⭐⭐⭐

#### 题目
```
numCourses = 4
prerequisites = [[1,0],[2,0],[3,1],[3,2]]
含义：要学课程 1 必须先学课程 0
问：能否完成所有课程？
```

#### 建模过程

```
1. 识别对象：课程 0,1,2,3
2. 识别关系：先修关系（依赖）
3. 建图：
   - 节点：{0, 1, 2, 3}
   - 有向边：0→1, 0→2, 1→3, 2→3
   - 含义：0 是 1 的前置，1 是 3 的前置
   
4. 问题转化：
   - 能否完成所有课程 → 图中是否有环
   - 有环 = A 依赖 B，B 依赖 A → 死锁 → 无法完成
```

#### 图示
```
    0
   / \
  v   v
  1   2
   \ /
    v
    3
    
无环 → 可以完成
```

#### 代码框架
```java
// 建图（邻接表）
List<List<Integer>> graph_breadth = new ArrayList<>();
for (int[] pre : prerequisites) {
    graph_breadth.get(pre[1]).add(pre[0]);  // pre[1] → pre[0]
}

// 检测环（DFS）
boolean hasCycle = false;
boolean[] visited = new boolean[n];
boolean[] onPath = new boolean[n];  // 记录当前路径

void dfs(int node) {
    if (onPath[node]) {
        hasCycle = true;  // 找到环
        return;
    }
    if (visited[node]) return;
    
    visited[node] = true;
    onPath[node] = true;
    
    for (int next : graph_breadth.get(node)) {
        dfs(next);
    }
    
    onPath[node] = false;  // 回溯
}
```

#### 举一反三
**类似问题**：
- 任务调度：任务 A 必须在任务 B 之前完成
- 软件依赖：安装包 A 前必须先安装 B
- 食谱制作：做蛋糕前要先准备材料

### 案例 3：岛屿数量（LeetCode 200）⭐⭐

#### 题目
```
给定网格，1 表示陆地，0 表示水
问：有多少个岛屿？
```

#### 建模过程

```
1. 识别对象：每个格子 (i,j)
2. 识别关系：相邻的陆地相连
3. 建图：
   - 节点：每个值为 1 的格子
   - 边：上下左右相邻的陆地
4. 问题转化：
   - 岛屿数量 = 连通分量数量
   - 用 DFS/BFS 遍历，每次遍历完一个连通分量，计数 +1
```

#### 图示
```
1 1 0 0 0
1 1 0 0 1    →  3 个岛屿
0 0 1 0 0
0 0 0 1 1
```

#### 代码框架
```java
int numIslands(char[][] grid) {
    int count = 0;
    for (int i = 0; i < grid.length; i++) {
        for (int j = 0; j < grid[0].length; j++) {
            if (grid[i][j] == '1') {
                count++;
                dfs(grid, i, j);  // 淹没整个岛屿
            }
        }
    }
    return count;
}

void dfs(char[][] grid, int i, int j) {
    // 边界检查 + 是否是陆地
    if (i < 0 || j < 0 || i >= grid.length || j >= grid[0].length 
        || grid[i][j] == '0') {
        return;
    }
    
    grid[i][j] = '0';  // 标记为已访问（淹没）
    
    // 四个方向
    dfs(grid, i-1, j);  // 上
    dfs(grid, i+1, j);  // 下
    dfs(grid, i, j-1);  // 左
    dfs(grid, i, j+1);  // 右
}
```

#### 举一反三
**类似问题**：
- 朋友圈（LeetCode 547）：M[i][j]=1 表示 i 和 j 是朋友，求朋友圈数量
- 水域大小：0 表示水域，求所有水域的大小
- 颜色填充：PS 油漆桶工具

### 案例 4：最小基因变化（LeetCode 433）⭐⭐

#### 题目
```
startGene = "AACCGGTT"
endGene = "AAACGGTA"
bank = ["AACCGATT","AACCGATA","AAACGGTA"]
每次只能变一个字符
问：最少需要几步？
```

#### 建模过程

```
1. 识别对象：每个基因序列
2. 识别关系：只差一个字符的两个序列可转换
3. 建图：
   - 节点：startGene + bank 中所有基因
   - 边：如果两个基因只差 1 个字符，连一条边
4. 问题转化：
   - 最少步数 = 最短路径长度
   - 用 BFS！BFS 第一次遇到 target 就是最短路径
```

#### 代码框架
```java
int minMutation(String start, String end, String[] bank) {
    Set<String> dict = new HashSet<>(Arrays.asList(bank));
    Queue<Pair<String, Integer>> queue = new LinkedList<>();
    queue.offer(new Pair<>(start, 0));
    Set<String> visited = new HashSet<>();
    
    while (!queue.isEmpty()) {
        Pair<String, Integer> curr = queue.poll();
        String gene = curr.getKey();
        int steps = curr.getValue();
        
        if (gene.equals(end)) return steps;
        
        // 尝试改变每个位置
        for (int i = 0; i < gene.length(); i++) {
            for (char c : "ACGT".toCharArray()) {
                String next = gene.substring(0,i) + c + gene.substring(i+1);
                if (dict.contains(next) && !visited.contains(next)) {
                    visited.add(next);
                    queue.offer(new Pair<>(next, steps + 1));
                }
            }
        }
    }
    return -1;
}
```

#### 举一反三
**类似问题**：
- 单词接龙（LeetCode 127）：hit→hot→dot→dog→cog
- 打开转盘锁（LeetCode 752）：每次转一个数字
- 迷宫最短路径

### 案例 5：重新安排行程（LeetCode 332）⭐⭐⭐

#### 题目
```
tickets = [["MUC","LHR"],["JFK","MUC"],["SFO","SJC"],["LHR","SFO"]]
从 JFK 出发，用完所有机票
问：字典序最小的行程
```

#### 建模过程

```
1. 识别对象：机场
2. 识别关系：机票表示有向边
3. 建图：
   - 节点：{JFK, MUC, LHR, SFO, SJC}
   - 边：JFK→MUC, MUC→LHR, LHR→SFO, SFO→SJC
4. 问题转化：
   - 用完所有边 → 欧拉路径
   - 字典序最小 → 优先走字典序小的边
   - 用 Hierholzer 算法（DFS 后序遍历）
```

#### 代码框架
```java
List<String> findItinerary(List<List<String>> tickets) {
    Map<String, PriorityQueue<String>> graph_breadth = new HashMap<>();
    for (List<String> ticket : tickets) {
        graph_breadth.computeIfAbsent(ticket.get(0), k->new PriorityQueue<>())
             .offer(ticket.get(1));
    }
    
    List<String> result = new ArrayList<>();
    dfs("JFK", graph_breadth, result);
    return result;
}

void dfs(String airport, Map<String, PriorityQueue<String>> graph_breadth, 
         List<String> result) {
    PriorityQueue<String> destinations = graph_breadth.get(airport);
    
    while (destinations != null && !destinations.isEmpty()) {
        String next = destinations.poll();
        dfs(next, graph_breadth, result);
    }
    
    // 后序遍历：当无路可走时才加入结果
    result.add(0, airport);
}
```

#### 举一反三
**类似问题**：
- 一笔画问题
- 邮递员送信
- 电路板布线

## 四、图的存储方式

### 1. 邻接表（推荐⭐⭐⭐）

```java
// 无权图
Map<String, List<String>> graph_breadth = new HashMap<>();
// 或者
List<List<Integer>> graph_breadth = new ArrayList<>();

// 有权图
Map<String, Map<String, Double>> graph_breadth = new HashMap<>();
// 或者
List<List<Edge>> graph_breadth = new ArrayList<>();

class Edge {
    int to;
    double weight;
}
```

**适用场景**：稀疏图（边数远小于 n²）

### 2. 邻接矩阵

```java
int[][] graph_breadth = new int[n][n];
// graph_breadth[i][j] = 1 表示 i→j 有边
// graph_breadth[i][j] = weight 表示权重
```

**适用场景**：稠密图、需要 O(1) 查询两点是否有边

### 3. 边列表

```java
List<int[]> edges = new ArrayList<>();
// edges.add(new int[]{u, v, weight});
```

**适用场景**：Kruskal 算法、只关心边的信息

## 五、常见图算法模板

### BFS 模板（最短路径）⭐⭐⭐

```java
int bfs(Node start, Node target) {
    Queue<Node> queue = new LinkedList<>();
    Set<Node> visited = new HashSet<>();
    
    queue.offer(start);
    visited.add(start);
    int steps = 0;
    
    while (!queue.isEmpty()) {
        int size = queue.size();
        for (int i = 0; i < size; i++) {
            Node curr = queue.poll();
            
            if (curr.equals(target)) {
                return steps;  // 找到目标
            }
            
            for (Node neighbor : curr.neighbors) {
                if (!visited.contains(neighbor)) {
                    visited.add(neighbor);
                    queue.offer(neighbor);
                }
            }
        }
        steps++;
    }
    return -1;
}
```

**适用问题**：
- 最短路径
- 最少步数
- 最小层数

---

### DFS 模板（遍历/搜索）⭐⭐⭐

```java
void dfs(Node node) {
    if (node == null) return;
    if (visited.contains(node)) return;
    
    visited.add(node);
    
    // 处理当前节点
    process(node);
    
    // 递归访问邻居
    for (Node neighbor : node.neighbors) {
        dfs(neighbor);
    }
}
```

**适用问题**：
- 连通分量计数
- 拓扑排序
- 环检测
- 所有路径搜索

### 并查集模板（连通性）⭐⭐

```java
class UnionFind {
    int[] parent;
    int count;  // 连通分量数量
    
    public UnionFind(int n) {
        parent = new int[n];
        for (int i = 0; i < n; i++) {
            parent[i] = i;
        }
        count = n;
    }
    
    public int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);  // 路径压缩
        }
        return parent[x];
    }
    
    public void union(int p, int q) {
        int rootP = find(p);
        int rootQ = find(q);
        if (rootP != rootQ) {
            parent[rootP] = rootQ;
            count--;
        }
    }
    
    public int getCount() {
        return count;
    }
}
```

**适用问题**：
- 连通分量数量
- 判断两个节点是否连通
- 动态连通性

## 六、做题 Checklist

看到题目后，按顺序问自己：

### Step 1: 识别阶段
- [ ] 是否涉及对象之间的关系？
- [ ] 是否需要从一个状态到另一个状态？
- [ ] 关系是否可以传递？

### Step 2: 建模阶段
- [ ] 什么是节点？（人、物、地点、状态...）
- [ ] 什么是边？（朋友关系、道路、转换规则...）
- [ ] 有向还是无向？
- [ ] 是否有权重？

### Step 3: 算法选择
- [ ] 求最短/最少 → BFS
- [ ] 求所有可能 → DFS
- [ ] 判断连通性 → BFS/DFS/并查集
- [ ] 检测环 → DFS/拓扑排序
- [ ] 最小生成树 → Prim/Kruskal
- [ ] 最短路径（带权）→ Dijkstra

### Step 4: 实现细节
- [ ] 用什么存储图？（邻接表/邻接矩阵）
- [ ] 如何标记已访问？（Set/boolean 数组）
- [ ] 需要记录什么信息？（步数、路径、父节点...）

## 七、易错点总结

### ❌ 错误 1：忘记标记已访问
```java
// 错误代码
void dfs(Node node) {
    for (Node neighbor : node.neighbors) {
        dfs(neighbor);  // 会无限循环！
    }
}

// 正确代码
void dfs(Node node) {
    visited.add(node);  // ✅ 先标记
    for (Node neighbor : node.neighbors) {
        if (!visited.contains(neighbor)) {
            dfs(neighbor);
        }
    }
}
```

### ❌ 错误 2：BFS 不记录步数
```java
// 需要在 Node 中封装步数信息
class State {
    Node node;
    int steps;
}
```

### ❌ 错误 3：建图时漏掉反向边
```java
// 无向图要加两条边
graph_breadth.get(u).add(v);
graph_breadth.get(v).add(u);  // ✅ 别忘了这个！
```

### ❌ 错误 4：并查集不路径压缩
```java
// 会导致超时
int find(int x) {
    if (parent[x] != x) {
        return find(parent[x]);  // ❌ 慢
    }
    return parent[x];
}

// 优化版本
int find(int x) {
    if (parent[x] != x) {
        parent[x] = find(parent[x]);  // ✅ 路径压缩
    }
    return parent[x];
}
```
